Summary
This video tutorial guides viewers through building a responsive front-end web project—an Expense Tracker application—to enhance their resumes with practical project experience. The project is divided into three parts: desktop version, mobile version (responsive design), and adding complete functionality. The focus here is on the first part: creating the desktop UI using HTML, CSS, Bootstrap, Google Fonts, and basic JavaScript setup.

Project Overview and Structure
Purpose: To create an attractive, functional expense tracker project that can be added to a resume, giving it more weight for job applications.

Project Parts:

Part 1: Desktop version UI (covered in this video)
Part 2: Mobile version for responsiveness
Part 3: Adding complete functionality (to be covered later)
Expense Tracker Features:

Add transactions with amount, date, and type (income/expense)
View total income, total expenses, and balance
Display all transactions in a table with delete functionality
UI Design Planning
Screens:
Home Page: Contains name input and login button
Main Screen: Header, form for adding transactions (amount, date, type), summary cards (total income, total expense, balance), and transactions table
UI Elements:
Buttons styled using Bootstrap classes with custom CSS overrides
Use of icons/images beside text (e.g., purse symbol, wallet icon)
Form fields include floating labels and input validation placeholders
Cards to display income, expenses, and balance with icons and styled backgrounds
Transaction table with columns: Amount, Transaction Type, Date, and Action (delete button)
Technical Setup & Tools
Development Environment: VS Code with Live Server extension
Libraries and Resources:
Bootstrap (CSS and JS) for responsive design and components
Google Fonts (Comfortaa font) for typography
Paper.js and other JS libraries (included but not deeply covered in this video)
Folder Structure: Separate folders and files for HTML, CSS (home.css, tracker.css), and images
Key CSS & HTML Implementation Details
Concept	Description
Parent Container Setup	Use of position: relative and fixed height (100vh) to contain child elements and manage overflow
Circle Design	A solid circle created via width/height: 750px; border-radius: 50%; background-color: black; position: absolute
Positioning	Absolute positioning with top, right properties and z-index to layer elements correctly
Flexbox Usage	display: flex with justify-content: space-between or space-around to align header elements and form items
Button Styling	Bootstrap classes like btn, btn-primary, btn-sm, with custom overrides for colors, padding, font-weight, shadow
Box Shadow	Applied to buttons and input fields using box-shadow: 0 4px 4px rgba(0,0,0,0.25) to add depth
Form Inputs	Floating labels, border-radius for rounded corners, consistent width (30%) for input fields, and drop-down for type
Cards	Bootstrap card components arranged in a flex row, styled with background colors, padding, font sizes, and icons
Table	Transaction table with header row styled differently, borders, padding, and box-shadow for neat appearance

Additional Implementation Notes
Use of Bootstrap margin utilities like m-0 to reset default margins
Custom padding and margin applied selectively to create balanced spacing
Google Fonts embedded via link tag for easy font application site-wide
The login button wrapped inside anchor tag to enable page navigation (redirect from login page to main tracker page)
Use of IDs and classes for targeted styling and JavaScript functionality preparation
Encouragement to experiment with styles such as box shadow opacity and colors for better UI refinement
Key Insights and Recommendations
Project divides complexity: Starting with UI layout before functionality helps understand structure and design flow.
Bootstrap integration: Using Bootstrap speeds up development and ensures responsiveness and consistency.
CSS positioning and z-index: Crucial for layering UI elements properly, especially for overlapping components.
Flexbox mastery: Essential for aligning and spacing elements dynamically.
Reusable components: Cards and form inputs styled with common classes for maintainability.
Use of Google Fonts: Enhances typography aesthetics effortlessly.
Incremental development: Encouraged to build desktop UI first, then move to responsive, then functionality.
Live preview with VS Code Live Server: Facilitates immediate feedback during development.
Documentation and community support: Telegram channel mentioned for sharing materials and doubt clearing.
Conclusion
This video thoroughly explains the front-end development process of an Expense Tracker app’s desktop UI using HTML, CSS, and Bootstrap. It covers the entire UI layout design from sketches to implementation, including forms, navigation, cards, and tables, with detailed CSS styling tips such as positioning, flexbox, and shadow effects. This project serves as practical experience for job seekers to strengthen their resumes by showcasing a complete, well-structured front-end application.
